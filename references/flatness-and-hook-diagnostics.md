# Flatness And Hook Diagnostics / 平淡与钩子诊断器

> 当读者反馈"读起来太平"、"前几章不抓人"、"没钩子"、"勾不住继续读"时，用本文件把模糊感受翻译成可定位、可修复的具体缺陷。
> Translates vague reader complaints into actionable, line-level defects.

适用时机：
- 审计章节时怀疑"叙事张力不足"
- 优化已有小说前的诊断步骤
- 设计开头三章前的预防检查
- 模型自检（每写完一章用本清单回扫）

---

## 第一部分：平淡诊断（Why It Reads Flat）

### A. 感官与具体物诊断

**症状**：场景读起来像 PPT 大纲，没有"在场感"。

**逐项检查（命中 ≥3 项 = 平）**：

- [ ] 第一段没有任何具体物体（家具、衣服、武器、食物、温度、光、声音）
- [ ] 整章只用了 1 种感官（基本只有视觉，听觉/触觉/嗅觉/味觉缺失）
- [ ] 出现"漂亮的房间"、"奢华的装饰"、"凌乱的摆设"等抽象形容，无具体物件
- [ ] 描写时间用"过了一会"、"许久之后"、"良久"，没有用具体物件标记时间流逝（茶凉了、香烧了、影子移了、雪积了）
- [ ] 角色情绪用形容词陈述（"她很愤怒"、"他十分悲痛"），无身体反应、无环境投射
- [ ] 整章没有一个"小到不重要却占住了画面"的细节（这是好作家的标志）

**修复方向**：每场景至少 2 种感官；每情绪至少一个身体反应或物件投射；每场景一个"无用但准确"的细节（手套上的污渍、桌角磕掉的漆、窗外有人吹口哨）。

---

### B. 句子节奏诊断

**症状**：读着读着犯困，句子像"叙述-叙述-叙述"。

**逐项检查**：

- [ ] 连续 5 句以上的句长方差小（都是 15-25 字之间）→ 缺节奏变化
- [ ] 段落开头主语连续 5 段相同（都是"他"或"她"或角色名）→ 视角僵
- [ ] 段落长度雷同（都是 4-6 行）→ 缺呼吸节奏
- [ ] 整章无一句"短句独立成段"（短句单段是节奏的鼓点）
- [ ] 整章无一句长句叠加复合从句（长句是情绪的延展）
- [ ] 对话比例 < 15%（场景死气）或 > 70%（缺画面感）
- [ ] 没有一处"动作打断对话"或"对话中插入观察"
- [ ] 时间副词重复（"然后"、"接着"、"于是"、"接下来"出现 ≥5 次）

**修复方向**：句长有意识交替（长-短-长长-短短）；段落长度刻意打乱；对话中插入动作和环境感知；用动作或物件代替"然后"做时间过渡。

---

### C. 信息揭露速度诊断

**症状**：剧情推进不波动，读者预测得到下一段。

**逐项检查**：

- [ ] 每段揭露的新信息量大致均匀（无信息密度峰谷）
- [ ] 没有"延迟揭露"——重要信息立刻给出，不留悬念
- [ ] 没有"加速揭露"——危机来临时仍按部就班描述
- [ ] 没有"反向揭露"——读者以为 A，结果是 B
- [ ] 章节结尾揭露完全（无未尽之意，无新疑问）
- [ ] 整章无"前面看似无关、后面回收"的细节
- [ ] 视角内信息和视角外信息完全一致（POV 没有"被欺骗"或"看错"过）

**修复方向**：刻意留白和延迟；危机段加速；至少一处反转（哪怕是小的）；每章结尾留新疑问；前期种"看似无关"的钉子。

---

### D. 内心起伏诊断

**症状**：人物在场景里情绪平直，没有"被推动"的感觉。

**逐项检查**：

- [ ] POV 在整场景情绪一致（全程平静或全程焦虑），无内心起伏
- [ ] 重要对话/动作之后没有内心余波
- [ ] 没有"想说但没说"、"想做但没做"的克制段
- [ ] 没有"误读对方"的段落
- [ ] POV 没有自我欺骗或被自己惊讶
- [ ] 整章没有一个被压下去的念头
- [ ] 没有身体先于意识的反应（手已经握紧了才意识到）

**修复方向**：参考 `hard-rules.md` 第 2 条 + `immersion-and-jargon-control.md` 的 interior beat 类型，每场景至少 2 种内心节拍。

---

### E. 对抗性诊断

**症状**：场景有事件但没有冲突感。

**逐项检查**：

- [ ] 场景里只有一种力量（POV 想做什么就能做，没有阻力）
- [ ] 阻力是抽象的（"局势险恶"、"形势严峻"），没有具体反对者或具体障碍
- [ ] POV 没有付出任何代价就达成目标
- [ ] 反对力量没有被人格化（没有具体的人、表情、话、动作来体现反对）
- [ ] 章节内 POV 没有任何选择困境（A 与 B 都不好选）
- [ ] 章节结束时没有"留下未解决的对抗"

**修复方向**：每场景明确 goal → obstacle → result；阻力人格化；至少一次选择困境；章末留未解决对抗。

---

### F. 段落开头与过渡诊断

**症状**：段与段之间的衔接平滑得没有惊喜。

**逐项检查**：

- [ ] 段落开头超过 60% 是主语 + 动词（"他抬起头"、"她转过身"）
- [ ] 段落开头超过 50% 是时间副词（"片刻后"、"过了一会"）
- [ ] 缺少"反向开头"（用环境、声音、感官、对方动作开始一段）
- [ ] 缺少"对话开头"（用一句话直接开段）
- [ ] 缺少"内心闪念开头"（用一个念头开段）

**修复方向**：段落开头方式至少 4 种轮换；用感官、对方动作、内心闪念、对话破开主语-动词的惯性。

## 第二部分：钩子诊断（Why The Hook Is Weak）

特别针对**前 3 章 / 前 1 万字**——网文环境下读者就用这点字数决定弃书。

### G. 第一章前 500 字诊断

> 前 500 字是"试读关"。读者在这里决定继续与否。

**必须命中（每项缺失 = P0）**：

- [ ] **具体物落点**：第一段有可视化的具体物（不能是"那是一个炎热的下午"这种抽象开头）
- [ ] **人物悬疑**：读完前 500 字，对主角至少有一个"想知道更多"的疑问（他为什么在这里？他在怕什么？他刚做了什么？）
- [ ] **当下张力**：前 500 字内有一个**当下**正在发生的紧张/危险/异常（不是回忆，不是设定介绍）
- [ ] **欲望或恐惧**：主角在前 500 字内表现出至少一项具体的欲望或恐惧（不是"她想过更好的生活"这种泛欲望）
- [ ] **不平衡感**：开场状态不是"日常稳定"，是**已经被打破的状态**

**禁止开头模式（命中 = 重写）**：

- [ ] 用主角自我介绍开头（"我叫 X，今年 X 岁，是 X 的女儿"）
- [ ] 用世界观/设定介绍开头（"在这个 X 的世界里……"）
- [ ] 用回忆/倒叙开头（除非是被一个**当下事件**触发的两三句闪回）
- [ ] 用气候/景物长篇描写开头（超过 100 字的纯环境）
- [ ] 用梦境开头（除非梦境内容直接关联当下危机）

---

### H. 前 3 章钩子密度诊断

每章必须命中（缺一项 = P1，缺两项 = P0）：

**第 1 章**：
- [ ] 立场景：读者知道地点、时间、氛围（通过具体物）
- [ ] 立人物：主角有一个**可记忆的具体瞬间**（不是档案介绍）
- [ ] 立矛盾：当下有一个正在发生的对抗或异常
- [ ] 立悬念：章末留至少一个未解疑问
- [ ] 立钩子：章末有一个让读者必须翻下一章的具体诱因（人来了、东西丢了、话没说完、门被推开）

**第 2 章**：
- [ ] 回应第 1 章的悬念，但**只解决一半**
- [ ] 引入新角色或新威胁
- [ ] 主角做一个**具体选择**（不能是被动反应）
- [ ] 揭露一个改变读者对主角理解的细节
- [ ] 章末再留新钩子

**第 3 章**：
- [ ] 主角的核心欲望/恐惧已经清晰
- [ ] 主要对抗力量已经被人格化（具体的敌人/障碍）
- [ ] 出现第一次**真正的代价**（损失、痛、决裂、暴露）
- [ ] 揭示一个让读者重新理解前两章的信息
- [ ] 章末让读者相信"后面会更精彩"

---

### I. 钩子类型诊断

**强钩子库**（每章末选 1-2 种，避免重复）：

- **身份钩子**：揭露/暗示主角或他人不为读者所知的身份
- **物件钩子**：一个物件出现/消失/异常
- **关系钩子**：人际关系发生急转
- **时间钩子**：明确的截止时间逼近（X 天后、明天早上、午夜之前）
- **空间钩子**：进入/逃离一个特定空间
- **信息钩子**：关键信息只揭示一半
- **威胁钩子**：具体的人/事物对主角产生威胁
- **诱惑钩子**：主角面对道德/欲望的两难
- **误读钩子**：主角对当前局势的理解明显不完整（读者比主角懂得多 / 主角比读者懂得多）
- **承诺钩子**：主角做出一个不可撤回的决定或承诺

**弱钩子（不要用）**：

- 模糊的"她有种不好的预感"
- 泛泛的"事情似乎并不简单"
- "这只是个开始"这类作者旁白
- 章末长篇内心总结（破坏节奏）

---

### J. 矛盾冲突强度诊断

**前 3 章必须出现的冲突层级**（缺即 P1）：

- [ ] **外部矛盾**：具体的人 vs 人 / 人 vs 事件
- [ ] **内部矛盾**：主角内心的两难、自我怀疑、欲望与恐惧的冲突
- [ ] **关系矛盾**：与亲密关系（家人、爱人、挚友、宿敌）的张力
- [ ] **价值矛盾**：主角的某个核心信念被挑战或动摇
- [ ] **环境矛盾**：世界规则与主角愿望的不兼容

**强度检查**：

- [ ] 每个冲突都是**具体的**（具体的人、具体的事、具体的代价）
- [ ] 冲突有**升级路径**（读者能感知它会变更糟）
- [ ] 主角对冲突有**真实的代价**（不是"如果失败可能会怎样"，是"已经在付代价"）
- [ ] 至少一个冲突主角**无法靠本能解决**（必须改变、学习、牺牲）

## 第三部分：综合诊断报告模板

诊断完一段/一章/前三章后，输出报告：

```markdown
# Diagnostic Report — [Target]

## 受检对象
- 类型：单章 / 前三章 / 段落 / 场景
- 字数：
- 文体：

## 平淡诊断结果
| 维度 | 命中数 | 严重度 | 主要问题 |
|---|---|---|---|
| A 感官与具体物 | x/6 | P0/P1/OK | ... |
| B 句子节奏 | x/8 | ... | ... |
| C 信息揭露速度 | x/7 | ... | ... |
| D 内心起伏 | x/7 | ... | ... |
| E 对抗性 | x/6 | ... | ... |
| F 段落开头与过渡 | x/5 | ... | ... |

## 钩子诊断结果（仅前三章/开头时填写）
| 维度 | 命中数 | 严重度 | 主要问题 |
|---|---|---|---|
| G 前 500 字 | x/5 必须项 | P0 if any miss | ... |
| H 前三章钩子密度 | x/15 | ... | ... |
| I 钩子类型 | 类型分布 | ... | ... |
| J 矛盾冲突强度 | x/5 层级 | ... | ... |

## 优先级修复清单（按严重度）
1. [P0] ...
2. [P0] ...
3. [P1] ...
4. [P2] ...

## 推荐手术粒度
- [ ] 句子级（< 5 处缺陷，局部加细节、调节奏）
- [ ] 段落级（5-15 处缺陷，重组段落、补内心）
- [ ] 场景级（>15 处或场景结构性缺陷，重写整段场景）
- [ ] 章节级（章节结构有问题，重写章节）

## 北极星问题
- 修复后这段是否会让读者**主动想读下一段**？
- 修复后是否每页都有至少一个"读者会停下来重读一遍"的瞬间？
```

---

## 使用方式

1. **生成新章节后自检**：用本清单回扫，命中 P0 立即修
2. **优化已有小说**：先跑一次完整诊断生成报告，再决定手术粒度（详见 `optimization-workflow.md`）
3. **设计开头时**：作为前置防御——开头三章按 G/H/I/J 的标准设计
4. **审计时**：与 `audit-protocol.md` 配合使用——本文件是更细的诊断层
